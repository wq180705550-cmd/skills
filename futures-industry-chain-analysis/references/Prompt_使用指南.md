# 商品期货产业链自动化分析 Prompt 使用指南

> 版本：v1.0 | 更新日期：2026-06-19
> 配套Prompt文件：`商品期货自动化分析Prompt_v1.md`

---

## 一、概述

本Prompt用于每日自动执行商品期货产业链分析，覆盖国内全部 76 个活跃品种（持仓量≥10,000手），通过多Agent辩论机制生成交易建议。

### 核心能力

| 能力 | 说明 |
|------|------|
| **品种覆盖** | 76个活跃品种，8大产业链 + 独立品种 |
| **数据源** | TqSdk（实时行情）+ AKshare（库存/结算）+ neodata（新闻）+ tdx（宏观） |
| **技术指标** | TqSdk内置70个指标，推荐组合12个 |
| **分析方法** | 产业链视角 + 多Agent辩论（11个角色） |
| **输出** | 产业链态势报告 + 交易策略（方向性/配对/跨产业链） |

### 硬规则

| 规则 | 要求 | 验证方法 |
|------|------|----------|
| 数据真实性 | 所有价格必须来自TqSdk实时查询 | 交叉验证 |
| 来源标注 | 每条结论标注数据来源 | 人工审查 |
| 缺失处理 | 标注"数据缺失"而非编造 | 自动检测 |
| 价格一致性 | 报告价格与TqSdk一致（±0.1%） | 自动比对 |
| 止损合理性 | 止损价必须低于入场价（做多） | 自动验证 |
| 风险收益比 | 目标收益/止损损失 ≥ 1.0 | 自动计算 |
| 产业链逻辑 | 交易建议必须有产业链逻辑支撑 | 人工审查 |

### 2.2 依赖项清单

| 依赖 | 类型 | 必选 | 状态 | 作用 | 不可用时影响 |
|------|------|:---:|:---:|------|-----------|
| **TqSdk** | Python包 | ✅ | ✅ 已就绪 | 实时行情、K线、技术指标 | **致命**：无法获取任何行情数据 |
| **TQ_USER/TQ_PASSWORD** | 环境变量 | ✅ | ✅ 已配置 | TqSdk认证 | **致命**：无法连接TqSdk |
| **trading-analysis** | WorkBuddy Skill | ✅ | ✅ 已安装 | 11个Agent多阶段辩论 | **严重**：无法生成交易建议 |
| **neodata** | WorkBuddy Skill | ❌ | ✅ 已就绪 | 新闻情绪、基本面 | 中等：缺少新闻维度 |
| **neodata token** | 缓存文件 | ❌ | ✅ 已配置 | neodata认证 | 中等：无法获取新闻 |
| **tdx-connector** | MCP连接器 | ❌ | ✅ 已连接 | 宏观数据、新闻 | 轻微：缺少宏观维度 |
| **AKshare** | Python包 | ❌ | ✅ 已安装 | 库存、结算价 | 轻微：缺少库存维度 |
| **industry-chain** | WorkBuddy Skill | ❌ | ✅ 已安装 | 产业链映射 | 轻微：可由Prompt内嵌逻辑替代 |

## 四、输出说明

### 4.1 输出文件

每次执行会生成以下文件：

```
F:\Traders\
├── analysis_report_YYYYMMDD_HHMMSS.json   # 结构化分析结果
├── analysis_report_YYYYMMDD_HHMMSS.md     # 可读版分析报告
└── analysis_YYYYMMDD_HHMMSS.log           # 执行日志
```

### 4.2 报告结构

```markdown
# 商品期货产业链分析报告
**日期**：YYYY-MM-DD
**数据时间**：HH:MM

## 一、市场概览
（8个产业链的龙头品种行情汇总表）

## 二、产业链态势
（每个产业链的价格传导、供需平衡、利润分析）

## 三、产业链交易策略
（每个产业链的整体方向、品种强弱、推荐方案）

## 四、风险提示
（宏观风险、产业链风险、品种特定风险）

## 五、数据来源
（各数据源的使用情况）
```

### 4.3 交易策略格式

每个产业链输出两种方案：

```
【黑色系产业链】
整体方向：偏多

品种强弱：rb(最强) > j(次强) > i(中性)

方案A（方向性）：做多黑色系，首选rb螺纹钢
  入场价：3131  目标价：3288  止损价：3037  仓位：5%

方案B（配对交易）：多rb空i，做多钢厂利润
  入场价差：2100  目标价差：2250  止损价差：2000  仓位：3%
```

## 五、常见问题

### Q1：TqSdk连接失败怎么办？

**症状**：提示"TQ_USER和TQ_PASSWORD环境变量未设置"或连接超时

**解决**：
1. 检查环境变量是否正确设置
2. 确认TqSdk账户是否有效
3. 检查网络连接

### Q2：neodata token过期怎么办？

**症状**：提示"TOKEN_EXPIRED"或"TOKEN_MISSING"

**解决**：
1. 系统会自动重新获取token
2. 如自动获取失败，告诉我"刷新neodata token"

### Q3：AKshare数据获取失败怎么办？

**症状**：报告中标注"库存数据缺失"

**解决**：
- 这是正常现象，AKshare的期货数据接口不稳定
- 库存数据是可选的，不影响核心分析
- 系统会自动降级，继续使用其他数据源

### Q4：如何调整分析范围？

**示例**：
```
只分析黑色系和能化产业链
排除金融期货
只分析持仓量前10的品种
```

### Q5：如何查看历史分析结果？

```
查看最近一次的分析报告
查看本周的分析报告
对比最近3天的分析结果
```

### Q6：如何调整交易策略参数？

```
将止损比例从3%调整为5%
将仓位从5%调整为3%
只输出方向性方案，不输出配对交易
```

## 六、Token消耗估算

| 分析范围 | 预估Token | 说明 |
|----------|:---------:|------|
| 单个产业链 | ~8K | 3个品种辩论 + 简要建议 |
| 3个产业链 | ~20K | 9个品种辩论 + 简要建议 |
| 全部8个产业链 | ~50K | 24个品种辩论 + 简要建议 |

**节省Token的技巧**：
1. 先用快速模式（只分析龙头品种）验证效果
2. 只分析感兴趣的产业链，不必每次都全覆盖
3. 使用简化版输出格式
