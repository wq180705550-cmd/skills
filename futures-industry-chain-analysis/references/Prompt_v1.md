# 商品期货产业链自动化分析 Prompt v1

> **用途**：部署为 WorkBuddy 自动化任务，每日收盘后自动执行
> **终止条件**：硬规则准确率=100%，幻觉率<5%，金融逻辑准确率≥70%
> **Token预算**：单次执行控制在 50K tokens 以内

---

## 一、总体工作流

```
Phase 1: 数据采集（并行） → Phase 2: 产业链聚类 → Phase 3: 多Agent辩论 → Phase 4: 风险裁决 → Phase 5: 输出报告
```

**核心原则**：
- 所有价格/持仓/成交数据**必须**来自 TqSdk 实时查询，禁止凭记忆或训练数据填写
- 每条结论必须标注数据来源，无法获取数据的维度标注"数据缺失"而非编造
- 分析以**产业链**为单位，不逐品种孤立分析
- **⚠️ 趋势判断必须基于技术指标**：高持仓 ≠ 趋势向上，禁止仅凭持仓量推测趋势方向
- **⚠️ K线必须使用主力连续合约**：格式为`KQ.m@EXCHANGE.product`，禁止使用具体到期合约

## 二、Phase 1：数据采集（并行执行，节省Token）

### 2.1 品种筛选

使用 TqSdk 获取所有持仓量 ≥ 10,000 手的活跃合约：
- 调用 `query_quotes(ins_class="FUTURE", expired=False)` 获取全品种列表
- 调用 `get_quote_list(symbols)` 批量获取行情
- 筛选条件：`open_interest >= 10000`
- 输出：品种代码、最新价、持仓量、成交量

### 2.2.1 产业链划分说明

**有色产业链**：
- SS(不锈钢)从黑色系移入有色，因为不锈钢的主要原料是镍
- ad(铝合金)从独立品种移入有色，因为铝合金和铝走势高度一致
- 辩论单元：ao(氧化铝) + al(铝) + ni(镍)，覆盖铝产业链和镍产业链

**化工产业链拆分**：
- **油化工**：以原油为原料的化工品，eg/eb/v/pp/l/PL/ps/bz
- **煤化工**：以煤为原料的化工品，MA(甲醇)/SH(烧碱)
- 辩论焦点：油化工关注烯烃利润，煤化工关注MTO/MTP价差

**橡胶产业链**：
- 从建材橡胶中独立出来
- ru(天然橡胶)和nr(20号胶)是天然橡胶
- br(丁二烯橡胶)是合成橡胶，受原油影响较大
- 辩论焦点：天然橡胶vs合成橡胶价差

**纸浆造纸产业链**：
- sp(纸浆)和op(双胶纸)形成完整产业链
- 辩论焦点：纸浆-纸张价差

**贵金属**：
- 新增pt(铂)，与au/ag同属贵金属

**建材产业链**：
- 从建材橡胶中独立出来，只包含FG(玻璃)、SA(纯碱)、UR(尿素)
- 辩论焦点：纯碱→玻璃成本传导

### 2.3 数据采集任务（每个产业链并行）

对每个产业链，采集以下数据：

**A. 行情数据（TqSdk，必采）**
- 主力合约最新价、持仓量、成交量
- **⚠️ K线必须使用主力连续合约**：`KQ.m@EXCHANGE.product`格式
- 最近 30-100 根日 K 线（用于计算技术指标）
- 计算：5日涨跌幅、持仓量变化率

**A2. 技术指标（TqSdk内置 ta 模块，必采）**

TqSdk 内置 70 个技术指标，直接调用 `tqsdk.ta` 模块计算，无需自行实现：

**推荐指标组合**（每个品种必算）：
1. **趋势判断**：MA5/MA10/MA20 + MACD + SAR
2. **动量判断**：RSI(14) + KDJ(9,3,3) + DMI(14,6)
3. **波动判断**：ATR(14) + BOLL(20,2)
4. **量能判断**：OBV + MFI(14)

**调用方式**：
```python

import tqsdk.ta as ta

api = TqApi(auth=TqAuth("账户", "密码"))

# 计算技术指标
ma5 = ta.MA(klines, 5)
ma10 = ta.MA(klines, 10)
ma20 = ta.MA(klines, 20)
macd = ta.MACD(klines, 12, 26, 9)
rsi = ta.RSI(klines, 14)
kdj = ta.KDJ(klines, 9, 3, 3)
atr = ta.ATR(klines, 14)
boll = ta.BOLL(klines, 20, 2)
dmi = ta.DMI(klines, 14, 6)
obv = ta.OBV(klines)
mfi = ta.MFI(klines, 14)
```

```

动量: RSI=58（中性偏强），KDJ金叉，DMI多头占优
波动: ATR=45（中等波动），BOLL中轨支撑
量能: OBV上升，MFI=62（资金流入）
综合判断: 偏多，但需关注RSI超买风险
```

## 三、Phase 2：产业链分析

### ⚠️ 趋势判断规则（必须遵守）

**禁止仅凭持仓量判断趋势**。高持仓只代表市场关注度高，不代表趋势方向。

**必须使用技术指标综合评分判断趋势**：

| 指标 | 多头条件 | 得分 | 空头条件 | 得分 |
|------|----------|:----:|----------|:----:|
| MA排列 | MA5>MA10>MA20 | +30 | MA5<MA10<MA20 | -30 |
| MACD | DIF>0 | +20 | DIF<0 | -20 |
| RSI | RSI<30(超卖反弹) | +10 | RSI>70(超买) | -10 |
| DMI | PDI>MDI | +20 | MDI>PDI | -20 |
| 价格位置 | 价格>MA20 | +15 | 价格<MA20 | -15 |

**趋势判断标准**：
| 得分范围 | 趋势判断 |
|:--------:|----------|
| ≥30 | 多头趋势 |
| 10~29 | 偏多震荡 |
| -9~9 | 震荡 |
| -29~-10 | 偏空震荡 |
| ≤-30 | 空头趋势 |

---

对每个产业链，生成一份**产业链态势报告**：

### 3.1 价格传导分析
- 上游原料价格变动方向 → 对下游成品的成本影响
- 计算：上游涨跌幅 vs 下游涨跌幅，判断传导是否顺畅
- **硬规则**：价格数据必须来自 Phase 1 的 TqSdk 查询结果

### 3.2 供需平衡判断
- 基于持仓量变化判断资金流向（增仓=分歧加大，减仓=趋势减弱）
- 基于库存数据判断供需紧张度（如可获取）
- 基于新闻判断供需预期变化

### 3.3 产业链利润分析
- 计算上下游价差（如螺纹-铁矿-焦炭价差）
- 判断产业链利润分配是否合理

## 四、Phase 3：产业链辩论（调用 trading-analysis skill）

**核心改变**：不是逐品种辩论，而是**按产业链关系辩论**。每个产业链选择 2-3 个关键品种（上游+下游），辩论焦点是产业链整体趋势和品种间相对强弱。

### 4.1 产业链辩论单元

每个产业链定义一个**辩论单元**，包含上下游关键品种：

| 产业链 | 辩论单元 | 辩论焦点 |
|--------|----------|----------|
| **黑色系** | (i铁矿 + j焦炭 + rb螺纹) | 成本推涨 vs 需求拉动？利润在上游还是下游？ |
| **能源链** | (sc原油 + lu低硫燃油 + bu沥青) | 裂解价差？原油-成品油传导效率？ |
| **聚酯链** | (PX对二甲苯 + TAPTA + PF短纤) | 聚酯利润？PX-PTA价差？ |
| **化工链** | (eg乙二醇 + ppPP + MA甲醇) | 烯烃利润？MTO/MTP价差？ |
| **有色** | (ao氧化铝 + al铝 + cu铜) | 氧化铝→铝的成本传导？铜铝比价？ |
| **贵金属** | (au黄金 + ag白银) | 黄金-白银比价，避险情绪传导 |
| **油脂油料** | (m豆粕 + y豆油 + p棕榈油) | 压榨利润？油脂间替代关系？ |
| **谷物软商品** | (c玉米 + CF棉花 + lh生猪) | 农产品供需周期？养殖利润？ |
| **建材化工** | (FG玻璃 + SA纯碱 + UR尿素) | 纯碱→玻璃成本传导？化工品需求？ |
| **金融期货** | (T国债 + IF股指) | 股债联动，利率-估值关系 |

### 4.2 辩论输入

对每个辩论单元，将以下数据打包传递给 trading-analysis skill：

- **品种行情数据**（来自 Phase 1）：所有品种的最新价、涨跌幅、持仓量、成交量
- **产业链态势报告**（来自 Phase 2）：价格传导分析、供需平衡判断、产业链利润分析
- **K线技术指标**（MA5/MA10/MA20/RSI/MACD）：每个品种的技术面
- **产业链关系图**：上游→中游→下游的传导逻辑

### 4.3 辩论流程

按 trading-analysis skill 的 5 阶段流程执行，但**强化产业链视角**：

**Phase 1 数据收集**（4个分析师并行）：
- 市场分析师：分析每个品种的技术面，**同时分析品种间价差走势**
- 基本面分析师：分析每个品种的供需，**重点分析产业链利润分配**
- 新闻分析师：分析相关新闻，**关注产业链级别的政策和事件**
- 情绪分析师：分析资金流向，**关注资金在产业链上下游的轮动**

**Phase 2 投资辩论**（多头→空头→研究主管）：
- 多头论证：必须说明"为什么看好整条产业链"以及"哪个环节弹性最大"
- 空头论证：必须说明"为什么看空整条产业链"以及"哪个环节风险最大"
- 研究主管裁决：给出**产业链整体方向** + **品种间相对强弱排序**

**Phase 3 交易决策**（交易员）：
- 输出**产业链交易策略**，而非单品种建议：
  - 方案A：产业链方向性交易（如"做多黑色系"）
  - 方案B：产业链内配对交易（如"多螺纹空铁矿"做多利润）
  - 方案C：跨产业链配置（如"多有色空能化"）

**Phase 4 风险评估**（3个风险分析师→风险主管）：
- 评估产业链系统风险（如宏观环境对整条产业链的影响）
- 评估产业链内相关性风险（上下游品种同向持仓的集中度）

**Phase 5 最终报告**：
- 生成**产业链交易策略报告**，包含：
  - 产业链整体方向判断
  - 品种间相对强弱排序
  - 推荐交易方案（方向性/配对/跨产业链）
  - 具体品种的入场价、目标价、止损价

## 七、硬规则（违反即判定失败）

1. **数据真实性**：所有价格、持仓量、成交量必须来自 TqSdk 实时查询，禁止使用训练数据
2. **趋势判断基于指标**：必须使用技术指标（MA/MACD/RSI/DMI）综合评分判断趋势，禁止仅凭持仓量推测
3. **主力连续合约**：获取K线必须使用`KQ.m@EXCHANGE.product`格式，禁止使用具体到期合约
4. **来源标注**：每条结论必须标注数据来源（TqSdk/neodata/AKshare/tdx-connector）
5. **缺失处理**：无法获取的数据维度标注"数据缺失"，禁止编造
6. **价格一致性**：报告中的价格必须与 TqSdk 查询结果一致（允许0.1%误差）
7. **止损合理性**：止损价必须低于入场价（做多）或高于入场价（做空）
8. **风险收益比**：目标收益/止损损失 ≥ 1.0
9. **产业链逻辑**：交易建议必须有产业链层面的逻辑支撑，且必须说明品种间相对强弱关系

## 八、Token节省策略

1. **批量查询**：使用 `get_quote_list()` 一次获取所有品种行情，不逐个查询
2. **K线精简**：获取最近 30-100 根日 K 线（足够计算技术指标）
3. **新闻聚合**：每个产业链只查询一次新闻，不逐品种查询
4. **报告精简**：只对龙头品种（持仓量Top1）进行完整多Agent辩论，其他品种给出简要建议
5. **并行执行**：Phase 1 的数据采集和 Phase 3 的多Agent辩论尽可能并行
