> 版本：v2.3 | 更新日期：2026-06-20
> 配套技能文件：`SKILL.md`

## 一、概述

本技能用于每日自动执行商品期货产业链分析，覆盖国内全部 76 个活跃品种（持仓量≥10,000手），通过多Agent辩论机制生成交易建议。

### 核心能力

| 能力 | 说明 |
|------|------|
| **品种覆盖** | 76+活跃品种，12大产业链 |
| **数据源** | TqSdk（实时行情）+ AKshare（库存/仓单/基差）+ neodata（新闻情绪）+ tdx-connector（宏观） |
| **技术指标** | MA/MACD/RSI/DMI/ATR/OBV，自适应权重系统 |
| **分析方法** | 产业链视角 + 多空辩论 + 风险评估 |
| **输出** | Markdown报告 + HTML可视化报告 |

### 版本演进

### 硬规则

| 规则 | 要求 | 验证方法 |
|------|------|----------|
| 数据真实性 | 所有价格必须来自TqSdk实时查询 | 交叉验证 |
| 来源标注 | 每条结论标注数据来源 | 人工审查 |
| 缺失处理 | 标注"数据缺失"而非编造 | 自动检测 |
| 价格一致性 | 报告价格与TqSdk一致（±0.1%） | 自动比对 |
| 止损合理性 | 止损价必须低于入场价（做多） | 自动验证 |
| 风险收益比 | 目标收益/止损损失 ≥ 1.0 | 自动计算 |
| 产业链逻辑 | 交易建议必须有产业链逻辑支撑 | 人工审查 |

## 三、使用方法

### 3.1 方式一：直接调用技能

在WorkBuddy对话框中输入：

```
使用 futures-industry-chain-analysis 分析商品期货
```

**试运行建议**（逐步扩大范围）：

```
# 第1次：只分析黑色系产业链
使用 futures-industry-chain-analysis 分析黑色系产业链

# 第2次：覆盖2-3个产业链
使用 futures-industry-chain-analysis 分析黑色系、能化、有色产业链

# 第3次：完整覆盖
使用 futures-industry-chain-analysis 执行完整分析
```

### 3.2 方式二：部署为自动化任务

确认手动运行效果后，可部署为每日自动执行的定时任务。

**部署步骤**：

1. 告诉我"部署为自动化任务"
2. 我会创建一个定时任务，配置如下：

```
名称：商品期货产业链每日分析
触发：每个交易日 15:30
工作目录：F:\Traders
```

3. 任务创建后会自动在每个交易日15:30执行

### 3.3 方式三：自定义参数运行

```
使用 futures-industry-chain-analysis 执行分析，参数如下：
- 覆盖产业链：黑色系、能化
- 分析模式：快速（只分析龙头品种）
- 输出格式：简化版
```

## 五、分析工作流

```
Phase 1 数据收集
  技术指标计算 + 产业链聚类 + 基本面数据收集
      ↓
Phase 2 多空辩论
  多头研究员 → 空头研究员 → 研究主管裁决
      ↓
Phase 3 交易决策
  交易员 → 入场价/目标价/止损价/仓位
      ↓
Phase 4 风险评估
  激进分析师 + 保守分析师 + 中性分析师 → 风险主管裁决
      ↓
Phase 5 整合报告
  Markdown报告 + HTML可视化报告
```

## 六、常见问题

### Q1：TqSdk连接失败怎么办？

**症状**：提示"TQ_USER和TQ_PASSWORD环境变量未设置"或连接超时

**解决**：
1. 检查环境变量是否正确设置
2. 确认TqSdk账户是否有效
3. 检查网络连接

### Q2：AKshare数据获取失败怎么办？

**症状**：报告中标注"库存数据缺失"

**解决**：
- 这是正常现象，AKshare的期货数据接口不稳定
- 库存数据是可选的，不影响核心分析
- 系统会自动降级，继续使用其他数据源

### Q3：如何调整分析范围？

**示例**：
```
只分析黑色系和能化产业链
只分析持仓量前10的品种
```

### Q4：如何查看历史分析结果？

```
查看最近一次的分析报告
查看本周的分析报告
对比最近3天的分析结果
```

### Q5：如何调整交易策略参数？

通过CONFIG_MANAGER配置调整：

```python
CONFIG_MANAGER['trading'] = {
    'entry_offset': 0.02,     # 入场偏移2%
    'target_return': 0.10,     # 目标收益10%
    'atr_multiplier': 2.0,     # ATR止损倍数
    'max_position': 8,         # 最大仓位8%
    'min_position': 2,         # 最小仓位2%
    'base_position': 5,        # 基础仓位5%
}
```
