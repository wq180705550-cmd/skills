# 自进化选股策略Skill

本Skill实现华泰证券金融工程团队提出的"自进化Skill"框架，将量化选股研究组织为可执行、可回溯、有约束的策略自进化流程。

## 核心思想

选股策略自进化的核心不是让模型无约束搜索参数，而是将投资逻辑、可调边界、样本隔离、版本保留和结果复盘写入Skill，使策略迭代在明确研究协议下持续推进。

### 三层框架

#### 1. 策略协议层（Strategy Protocol）

明确策略可以怎样变化，包括：

- **策略内核**：不可偏离的投资假设
- **数据与因子边界**：可使用因子、方向定义和派生口径
- **组合约束**：股票池、调仓频率、持仓数量、单票上限、交易成本
- **迭代边界**：优先通过配置入参调整，原则上不修改公共代码

#### 2. 版本管理层（Version Management）

回答每一次变化如何被记录和复盘：

- 每个版本独立目录
- 保存配置文件、组合权重、打分记录、调仓日期、持仓明细、分样本回测结果
- 保留/回滚机制
- 版本日志：假设、修改项、训练集/验证集证据、失败归因

#### 3. 样本隔离层（Sample Isolation）

回答策略是否真的变好，而不是变得更贴合历史：

- **训练集**（Training Set）：用于提出和筛选策略假设
- **验证集**（Validation Set）：用于判断某一版本是否值得保留
- **测试集**（Test Set）：只用于最终人工复核，不进入自动调参闭环

## 策略说明：稳健低波价值优选

### 投资逻辑

在具备基本盈利能力和可交易性的股票中，先判断其是否具备相对估值安全边际，再结合分红质量、现金流防守、盈利稳定性和低波特征进行综合筛选。

**不是简单买入股息率最高或历史波动最低的股票**，而是：

1. 确认公司具备正盈利和估值数据
2. 在行业和盈利能力可比的条件下寻找估值更有吸引力的股票
3. 叠加质量、分红、成长稳定和风险约束构建组合

### 策略流程

#### 第一步：构建可选池

- **基础股票池**：全A股
- **可交易性过滤**：
  - ROE为正
  - 估值因子可得
  - 流通市值和成交额数据无缺失
  - 处于可交易状态
- **流动性过滤**：
  - 流通市值分位数阈值：默认0.35
  - 20日成交额分位数阈值：默认0.40
  - 若候选数量不足，则回退至较宽松阈值

#### 第二步：构建价值基础池

- **价值因子**：EP_TTM（市盈率倒数的TTM版本）
- **盈利能力锚**：ROE_TTM
- **行业内估值比较**：在每个调仓日用行业内回归方式刻画估值与ROE之间的关系
- **价值残差**：根据残差识别在相似盈利能力下估值更具安全边际的股票
- **保留阈值**：价值残差排序较优的三分之一股票作为基础池（默认）
- **容错机制**：若基础池数量不足，则使用较宽松阈值保证组合可构建

#### 第三步：计算三组得分

在价值基础池内计算三组得分，共同决定选股排序：

**leader组（1/3权重）**：龙头质量
- ROIC_TTM
- 收入规模代理
- 行业内排序

**dividend组（1/3权重）**：分红和现金流防守能力
- 两年平均股息收益
- 两年平均股利价格比
- 经营现金流收益
- 60日波动率
- 行业内排序

**growth_stability组（1/3权重）**：成长稳定性
- 净利润增速
- 利润增长稳定性
- 行业内排序

#### 第四步：组合构建

- **调仓频率**：按月调仓
- **目标持仓**：30只（默认）
- **缓冲池**：45只（默认）
- **单票权重上限**：5%
- **定权方式**：使用下行波动率作为风险因子进行倒数定权
  - 使低下行风险股票获得更高权重
  - 通过单票上限控制集中度
- **仓位管理**：初始版本不加入市场择时或仓位择时，剩余优化应主要围绕红利、防守、价值安全边际和低波风险控制展开

### 迭代约束

- **默认原则**：不修改已有策略代码、回测代码和其他工具代码，除非发现确定性错误
- **每轮迭代**：围绕一个主要假设展开
- **有效改进方向**：
  - 红利防守权重调整
  - 价值基础池阈值调整
  - 组合缓冲持仓数调整
  - 个股权重上限调整
  - 流动性约束调整
- **无效尝试**：复杂因子替换或偏离原始逻辑的尝试并未稳定进入优势版本

## 使用工作流

### 工作流1：初始策略构建

1. 准备数据：股票池、因子数据、价格数据
2. 配置skill_config.yaml：
   - 股票池参数
   - 因子组定义
   - 组合规则
   - 可调参数范围
3. 运行策略生成脚本：读取配置，生成持仓
4. 运行回测脚本：读取持仓，完成分样本回测
5. 查看初始版本结果：年化收益率、波动率、夏普比率、最大回撤、换手率

### 工作流2：策略迭代优化

1. 提出改进假设：基于初始版本结果，提出一个主要改进方向
2. 修改skill_config.yaml：调整对应参数
3. 运行策略生成和回测脚本
4. 在验证集上评估：是否改善
5. 若改善则保留版本，若无效则回滚
6. 继续下一轮实验
7. 最终在测试集上人工复核

### 工作流3：版本复盘

1. 查看version文件夹：每个版本的配置、权重、得分、调仓日和回测结果
2. 分析改进路径：从保留版本路径看，策略改进主要来自多次小幅、可解释的配置调整
3. 识别有效方向：红利防守权重、价值基础池阈值、组合缓冲持仓数、个股权重上限、流动性约束
4. 避免无效尝试：复杂因子替换或偏离原始逻辑的尝试

## 配置文件说明

### skill_config.yaml

可机器读取的参数配置，包括：

```yaml
# 股票池参数
stock_pool:
  base_pool: "全A股"
  roe_threshold: 0  # ROE为正
  market_cap_percentile: 0.35  # 流通市值分位数阈值
  turnover_percentile: 0.40  # 20日成交额分位数阈值

# 因子组定义
factors:
  value:
    name: "EP_TTM"
    description: "市盈率倒数的TTM版本"
  profitability:
    name: "ROE_TTM"
    description: "净资产收益率TTM"
  quality:
    name: "ROIC_TTM"
    description: "投入资本回报率TTM"
  dividend:
    names: ["两年平均股息收益", "两年平均股利价格比", "经营现金流收益"]
  volatility:
    name: "60日波动率"
    description: "用于衡量低波特征"

# 组合规则
portfolio:
  rebalance_frequency: "M"  # 按月调仓
  target_holdings: 30
  buffer_pool: 45
  max_weight: 0.05  # 单票权重上限5%
  weighting_method: "下行波动率倒数定权"

# 可调参数（迭代边界）
tunable_params:
  dividend_weight:
    default: 0.33
    range: [0.20, 0.50]
    description: "红利防守权重"
  value_pool_percentile:
    default: 0.33
    range: [0.25, 0.50]
    description: "价值基础池阈值（保留较优的百分比）"
  buffer_pool_size:
    default: 45
    range: [35, 60]
    description: "组合缓冲持仓数"
  max_weight:
    default: 0.05
    range: [0.03, 0.08]
    description: "个股权重上限"
  liquidity_threshold:
    default: 0.40
    range: [0.30, 0.50]
    description: "流动性约束阈值"
```

## 脚本说明

### scripts/strategy_gen.py

读取skill_config.yaml配置，生成持仓。

**功能**：
- 构建可选池（流动性过滤）
- 构建价值基础池（行业内回归残差）
- 计算三组得分（leader、dividend、growth_stability）
- 组合构建（下行波动率倒数定权）
- 输出持仓明细

### scripts/backtest.py

读取持仓并完成分样本回测。

**功能**：
- 分样本回测（训练集/验证集/测试集）
- 计算绩效指标（年化收益率、波动率、夏普比率、最大回撤、换手率）
- 输出净值曲线
- 保存版本结果

### scripts/version_manager.py

管理版本迭代和复盘。

**功能**：
- 创建新版本目录
- 保存版本配置和结果
- 判断是否保留版本（基于验证集指标）
- 版本复盘和可视化

## 风险提示

- 大模型是海量数据训练获得的产物，输出准确性及稳定性可能存在风险
- 不同大模型效果存在差异，需谨慎选择
- 非本地大模型处理敏感数据或有信息泄露风险
- 策略回测结果不代表实际投资收益
- 历史表现不代表未来收益
