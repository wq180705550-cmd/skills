# EmbodiSkill v2.0：面向具身技能自演化的技能感知反思与进化

> **演化说明**：本 skill 经 SkillEvolver 三阶段演化（探索→反思→审计），新增协同使用模式、双层结构编写规范、决策流程图等增强内容

---

## 技能结构（双层规范）

```
EmbodiSkill = (S_body 技能主体, S_appendix 技能附录)
```

- **S_body（技能主体）**：核心规范 —— 四种反思类型、执行协议、设计原则
- **S_appendix（技能附录）**：强调标记 —— 判断标准、常见失误、必须执行的步骤

---

# S_body：技能主体

## 核心概念

### 什么是技能？

**技能（Skill）** 是一种持久且可修订的过程规范，用于指导智能体跨任务执行。它由两部分组成：

```
技能 S = (技能主体 S_body, 技能附录 S_appendix)
```

- **S_body（技能主体）**: 主要的规范性内容 —— 规则、流程、策略、前置条件、子目标排序、恢复策略
- **S_appendix（技能附录）**: 强调标记，用于突出 S_body 中智能体容易忽视或未能遵循的有效内容。**不引入新规则**

### 核心问题

现有的技能演化方法采用**粗粒度的全技能更新**：轨迹 → 通用反馈 → 重写整个技能。这种方式存在问题：

1. **失败的轨迹** 不一定意味着技能错误 —— 智能体可能只是未能遵循有效指导（**执行失误**）
2. 粗粒度更新会**覆盖有效内容**、**累积冗余**、**保留错误规定**

### EmbodiSkill 解决方案

**技能感知反思**：不将轨迹转换为通用反馈，而是针对**当前技能**审视每条轨迹，以确定：
- 哪些技能内容应该**添加**（发现）
- 哪些应该**优化**（优化）
- 哪些是**错误/不完整**的（技能缺陷）
- 哪些是**有效但未遵循**的（执行失误）

然后应用**精准修订** —— 只改变应该改变的，保留有效的内容。

## 四种反思类型

### 决策流程图

```
                    轨迹结果
                       │
           ┌───────────┴───────────┐
           │                       │
        成功(r=1)               失败(r=0)
           │                       │
     ┌─────┴─────┐           ┌─────┴─────┐
     │           │           │           │
   发现      优化       技能缺陷    执行失误
 (DISCOVERY) (OPTIMIZATION) (DEFECT)   (LAPSE)
     │           │           │           │
     ▼           ▼           ▼           ▼
  添加新内容  修改现有内容  修正错误内容  添加强调项
  到 S_body   在 S_body    在 S_body    到 S_appendix
```

### 1. 发现（DISCOVERY）—— 新技能内容

**触发时机**: 成功的轨迹揭示了当前技能未涵盖的有用过程知识。

| 字段 | 描述 |
|---|---|
| **触发条件** | 任务成功（r=1）且轨迹包含当前 S_body 中不存在的可复用模式 |
| **判断标准** | 新模式是否可复用？是否属于当前技能范围？ |
| **目标** | 无（提出新内容） |
| **指令** | 指定要添加的新技能内容 |
| **主体影响** | 向 S_body **添加**新内容 |
| **附录影响** | 无 |

### 2. 优化（OPTIMIZATION）—— 更好的执行方式

**触发时机**: 成功的轨迹展示了执行现有技能内容的更有效方式。

| 字段 | 描述 |
|---|---|
| **触发条件** | 任务成功（r=1）且轨迹展示了优于现有 S_body 内容的改进方式 |
| **判断标准** | 改进是否显著？是否保持原有逻辑？ |
| **目标** | 特定的现有 S_body 内容（b_i） |
| **指令** | 目标内容的修订版本或修改指令 |
| **主体影响** | **修改**现有 S_body 内容 |
| **附录影响** | 无 |

### 3. 技能缺陷（SKILL DEFECT）—— 错误/不完整的技能内容

**触发时机**: 失败的轨迹暴露出当前技能内容存在错误、不完整或规定不足。

| 字段 | 描述 |
|---|---|
| **触发条件** | 任务失败（r=0）且失败由错误/不完整/规定不足的技能内容导致 |
| **判断标准** | 技能内容是否逻辑错误？是否遗漏关键步骤？ |
| **目标** | 特定的有问题的 S_body 内容（b_i） |
| **指令** | 缺陷的轨迹证据 + 修正建议 |
| **主体影响** | **修正**现有 S_body 内容 |
| **附录影响** | 无 |

### 4. 执行失误（EXECUTION LAPSE）—— 有效技能，智能体未遵循

**触发时机**: 失败的轨迹中技能内容是**有效**的，但智能体偏离或忽视了它。

| 字段 | 描述 |
|---|---|
| **触发条件** | 任务失败（r=0）且失败由智能体**未遵循**有效技能指导导致 |
| **判断标准** | 技能内容是否正确？智能体是否偏离了指导？ |
| **目标** | 被违反的特定有效 S_body 内容（b_i） |
| **指令** | 描述技能要求与实际执行之间的偏差 |
| **主体影响** | **无** —— S_body 被保留 |
| **附录影响** | 向 S_appendix 添加强调项，提醒智能体遵循此内容 |

│                                                             │
│   执行 ──▶ 反思 ──▶ 累积 ──▶ 整合 ──▶ 修订主体 ──▶ 更新附录   │

### 关键参数

| 参数 | 符号 | 描述 | 默认值 |
|---|---|---|---|
| **每条轨迹最大反思数** | K | 每条轨迹的最大反思记录数 | 3 |
| **修订间隔** | B | 技能修订前累积的反思数量 | 10 |

## 执行协议

### 阶段 1：技能引导执行
- 将 S_body 和 S_appendix 作为过程指导呈现给执行器
- 执行器生成动作，记录完整轨迹
- 执行器参数保持固定 —— 所有改进通过技能演化实现

### 阶段 2：技能感知反思
- 分析轨迹结果（成功 r=1 或失败 r=0）
- 对每个可识别的反思点（最多 K 个）进行分类
- 若轨迹未提供可靠的修订证据，返回空集

### 阶段 3：反思累积与分区
- 将新反思添加到缓冲区
- 达到修订间隔 B 后，按类型分区：发现 / 优化 / 技能缺陷 / 执行失误

### 阶段 4：整合与修订
- **整合主体级修订**：移除冗余、合并重叠、解决冲突
- **修订技能主体**：发现→添加、优化→修改、技能缺陷→修正
- **更新技能附录**：将执行失误转换为强调项

## 双场景适配

### 具身智能体场景
- **环境**: 物理或物理基础的 3D 环境（ALFWorld、EmbodiedBench 等）
- **技能内容**: 对象搜索策略、动作前置条件、容器状态检查、导航模式
- **关键挑战**: 感知噪声、空间定位、对象状态变化

- **环境**: 数字环境（代码编辑器、文件系统、API、网页）

- **关键挑战**: 上下文窗口限制、工具可用性、API 变化

## 关键设计原则

1. **选择性修订优于粗粒度重写** —— 仅修改有基于证据的正当理由的技能内容
2. **主体与附录分离** —— 主体变更需要发现/优化/技能缺陷证据；附录变更仅来自执行失误证据
3. **零训练操作** —— 无梯度更新，所有改进外部化到演化的技能中
4. **渐进式累积** —— 技能随时间变得更加完整、准确和可执行
5. **基于证据的反思** —— 每次修订必须引用具体的轨迹证据
6. **与 SkillEvolver 协同** —— 冷启动用 SkillEvolver，热迭代用 EmbodiSkill

---

## 反模式

- 不要将所有失败轨迹都视为技能缺陷的证据
- 不要修改技能内容而不引用轨迹证据
- 不要通过附录引入新规则
- 不要跳过整合步骤
- 不要基于执行失误反思修订技能主体
- **不要混淆技能缺陷与执行失误的判断标准**
- **不要跳过与 SkillEvolver 的协同审计**

# S_appendix：技能附录

> **重要提示**：本附录包含关键约束和常见失误，使用本 skill 时必须严格遵守

## 【必须执行】关键步骤

### 1. 反思分类（不可跳过）
- [ ] 确定轨迹结果（成功/失败）
- [ ] 判断技能内容是否正确
- [ ] 判断智能体是否遵循技能
- [ ] 分类为四种反思类型之一

### 2. 判断标准检查
- [ ] **发现**：成功 + 新模式 + 可复用
- [ ] **优化**：成功 + 改进现有内容
- [ ] **技能缺陷**：失败 + 技能内容错误
- [ ] **执行失误**：失败 + 技能正确但未遵循

### 3. 修订执行
- [ ] 发现/优化/缺陷 → 修订 S_body
- [ ] 执行失误 → 更新 S_appendix
- [ ] 达到修订间隔后整合

## 【常见失误】执行失误警示

### ❌ 失误1：混淆技能缺陷与执行失误
**后果**：错误修改有效技能内容 / 忽略真实技能问题
**判断标准**：
- 技能内容逻辑错误？→ 技能缺陷
- 技能内容正确但智能体偏离？→ 执行失误
**修正**：仔细分析失败根因

### ❌ 失误2：通过附录引入新规则
**后果**：破坏主体与附录的分离原则
**修正**：附录只能添加强调项，不能引入新规则

### ❌ 失误3：跳过整合步骤
**后果**：冗余、冲突、重叠的技能内容
**修正**：修订前必须整合主体级修订

### ❌ 失误4：修改技能无轨迹证据
**后果**：主观修改，无法验证
**修正**：每次修订必须引用具体轨迹证据

### ❌ 失误5：忽略与 SkillEvolver 的协同
**后果**：缺少独立审计，可能部署失败
**修正**：定期使用 SkillEvolver 进行独立审计

## 【强调标记】关键约束

> ⚠️ **警告**：以下约束必须严格遵守，否则可能导致严重后果

1. **禁止**混淆技能缺陷与执行失误的判断标准
2. **禁止**通过附录引入新规则（只能强调）
3. **禁止**跳过反思累积与整合步骤
4. **禁止**无轨迹证据修改技能内容
5. **禁止**基于执行失误修订技能主体
6. **禁止**长期不进行 SkillEvolver 独立审计
