# S_body：技能主体

## 核心概念

### 从传统 AI 到 Agentic AI

| 维度 | 传统 AI | Agentic AI |
|---|---|---|
| **工作模式** | 被动分析引擎，依赖人类提示 | 自主、目标驱动的系统 |
| **核心能力** | 模式识别、预测、分类 | 感知环境、多步推理、执行动作、从结果中学习 |
| **人类介入** | 需要持续提示和预定义规则 | 最小化人类干预 |
| **适应性** | 静态规则，需人工更新 | 动态学习，自主更新内部记忆 |

### 因子投资的核心挑战

1. **"因子动物园"问题**：潜在的价格-成交量组合维度远超人类认知能力
2. **数据挖掘偏差**：回测过拟合、p-hacking、虚假发现
3. **研究效率瓶颈**：人工特征工程、假设定义、持续提示
4. **技能演化缺失**：静态规则无法自适应市场变化
5. **审计门控不足**：缺乏系统化的过拟合/泄露检测

### Agentic AI + 技能演化解决方案

将 Agentic AI 作为**自主量化研究员**，结合**技能感知反思**与**独立审计门控**：
- 给定高层目标（如最大化风险调整收益、控制换手率和交易成本）
- 独立导航广阔的假设空间
- 使用**多策略探索**自主构建因子表达式
- 对历史市场数据进行回测
- 评估经济逻辑
- 基于实证反馈动态更新内部记忆
- 通过**技能感知反思**持续改进因子发现技能本身

## 因子发现流程

### 阶段一：策略多样化探索（Strategy Diversification）

**目的**：用不同策略执行因子生成，最大化覆盖不同的成功/失败模式

```
输入: 当前记忆、高层目标、探索次数 K（默认 4）
输出: 因子候选集 {f_{r,i}}, 评估指标 {Sharpe_{r,i}, IC_{r,i}}

协议:
1. 对每次探索 i = 1, ..., K:
   a. 从策略池中选择探索策略 s_i:
      - s_1: 符号回归 (Symbolic Regression)
      - s_2: 遗传编程 (GP)
      - s_3: 强化学习 (RL)
      - s_4: LLM启发式生成
   b. 使用当前记忆生成新因子假设
   c. 记录完整因子轨迹 τ_{r,i} 和评估指标
2. 确保策略多样性:
   - 不同的生成路径覆盖不同的因子类型
   - 成功和失败因子都产生有价值的信号
   - 避免所有尝试都陷入相同的局部最优
```

### 阶段二：技能感知反思与对比更新

**目的**：基于成功/失败因子的对比分析，生成精准的技能修订

**核心原则**：
- 学习信号来自**部署后另一个智能体使用技能时的行为**
- 不是作者智能体的自我反思
- 候选技能可能因以下原因失败:
  * 产生错误信号（未来函数、数据窥探）
  * 遗漏必要指令（缺少风控、未处理异常）
  * 暴露误导性流程（错误的参数范围）
  * 被使用智能体**静默绕过**（内容有效但运行时从未被调用）

#### 四种反思类型应用于因子发现

| 反思类型 | 触发条件 | 主体影响 | 附录影响 |
|----------|----------|----------|----------|
| **发现(DISCOVERY)** | 成功因子揭示了当前技能未涵盖的有用模式 | 向 S_body 添加新内容 | 无 |
| **优化(OPTIMIZATION)** | 成功因子展示了优于现有技能内容的改进方式 | 修改现有 S_body 内容 | 无 |
| **技能缺陷(SKILL DEFECT)** | 失败因子暴露出技能内容存在错误/不完整 | 修正现有 S_body 内容 | 无 |
| **执行失误(EXECUTION LAPSE)** | 失败因子中技能有效，但智能体未遵循 | 无 | 向 S_appendix 添加强调项 |

**对比分析流程**：

1. **分析成功因子 F+**:
   - 识别成功因子的关键特征和逻辑模式
   - 提取可复用的过程知识
   - 标记技能中已覆盖和未覆盖的部分

2. **分析失败因子 F-**:
   - 定位失败根因（技能缺陷 vs 执行偏差）
   - 识别技能中缺失、错误或误导的内容
   - 检测是否存在"静默绕过"

3. **生成对比修订**:
   - 添加缺失的必要指令
   - 修正错误或误导的过程描述
   - 强化被绕过的关键步骤
   - 移除导致混淆的冗余内容

**技能感知演化螺旋**：
```
执行因子 → 技能感知反思 → 累积 → 整合 → 修订主体 → 更新附录 → 重复
```

**关键参数**：

| 参数 | 符号 | 描述 | 默认值 |
|------|------|------|--------|
| **每条轨迹最大反思数** | K | 每个因子的最大反思记录数 | 3 |
| **修订间隔** | B | 技能修订前累积的反思数量 | 10 |

### 阶段三：独立审计与定稿（Independent Audit）

**目的**：在独立新会话中审计候选因子，防止泄露、过拟合和部署特有失败

**审计检查清单（Audit Checklist）**：

| 检查项 | 说明 | 通过标准 |
|--------|------|----------|
| **1. 数据泄露检测** | 因子是否包含未来信息？是否编码了不应包含的测试期信息？ | 样本外 IC > 样本内 IC × 0.7 |
| **2. 过拟合检测** | 因子是否过度特化于训练期？移除训练特定细节后是否仍有效？ | 跨期稳定性 ICIR > 0.5 |
| **3. 静默绕过检测** | 技能内容是否有效但在运行时从未被调用？ | 审计智能体实际使用了技能 |
| **4. 部署失败检测** | 技能是否导致使用智能体产生新的失败模式？ | 实盘模拟无异常 |
| **5. 功能完整性** | 技能是否包含执行所需的全部必要信息？ | 检查清单 100% 通过 |

**静默绕过（Silent Bypass）检测机制**：
- 技能在内容上看起来有效（指令正确、流程合理）
- 但使用智能体在运行时**从未实际调用/依赖该技能**
- 这种失败从探索轨迹中不可见，只有通过部署交接才能检测

## 因子聚合与组合构建

### 信号合成

```
方法 1: 简单线性组合
  signal = w1 * f1 + w2 * f2 + ... + wn * fn
  权重: 基于历史夏普比率或信息比率

方法 2: 非线性聚合（LightGBM）
  优势: 捕捉因子间复杂交互
  正则化: 防止过拟合
  样本外保护: 早停机制
```

### 组合构建

```
步骤 1: 信号标准化
  - 截面排序
  - Z-score 标准化
  - 极值处理

步骤 2: 预期收益代理
  - 信号强度映射
  - 置信度加权

步骤 3: 十分位组合形成
  - 按信号强度排序
  - 等权重或市值加权

步骤 4: 多空组合
  - 做多 top decile
  - 做空 bottom decile
  - 计算多空收益差
```

## 过拟合缓解机制

### 1. 经济正则化

```
要求: 每个因子必须有清晰的经济学解释
检验:
- 是否与已知风险溢价相关？
- 是否捕捉特定的市场摩擦？
- 是否反映投资者行为偏差？
```

### 2. 统计门槛

```
严格统计检验:
- t 统计量 > 3.0（对应 p < 0.01）
- 样本外 R² > 0
- 信息比率 > 0.5
- 夏普比率 > 1.5
```

### 3. 多重检验校正

```
问题: 测试大量因子时，虚假发现概率上升
解决方案:
- Bonferroni 校正: 调整显著性水平
- FDR 控制: 限制假阳性比例
- 有效因子数估计: 考虑因子相关性
```

### 4. 样本外衰减检验

```
检验因子在不同时期的稳定性:
- 训练期 vs 测试期绩效对比
- 滚动窗口绩效追踪
- 衰减率阈值（建议 < 30%）
```

### 5. 时间与策略隔离

```
严格时间分割:
- 训练期: 用于因子发现
- 验证期: 用于因子筛选
- 测试期: 最终绩效评估（仅使用一次）

策略隔离:
- 不同因子使用独立数据路径
- 避免信息泄露
```

## 污染控制机制

### 第一层：严格的训练/测试分割
- 训练期因子在源处被标记，测试期不可见
- Agentic AI 仅能访问训练期数据
- 测试期 oracle 不能泄露到评估制品中

### 第二层：审计隔离
- 审计智能体在独立会话中运行
- 无法访问作者智能体的上下文
- 通过 PreToolUse 钩子强制执行数据隔离

## 关键设计原则

### 1. 自主性 > 被动性
- 智能体自主导航假设空间，而非依赖人工提示
- 动态更新记忆，而非静态规则

### 2. 可解释性 > 黑箱
- 每个因子必须有经济逻辑支撑
- 符号回归确保数学表达式可解释

### 3. 样本外验证 > 样本内拟合
- 严格的样本外测试
- 多重检验校正防止虚假发现

### 4. 经济逻辑 > 数据挖掘
- 经济正则化约束
- 行为金融理论支撑

### 5. 实用性 > 理论性
- 考虑交易成本、换手率、容量约束
- 机构可执行的策略

### 6. 技能演化 > 静态规则（EmbodiSkill + SkillEvolver 贡献）
- 技能感知反思持续改进
- 独立审计门控保障质量
- 策略多样化探索避免局部最优

# S_appendix：技能附录

> **重要提示**：本附录包含关键约束和常见失误，使用本 skill 时必须严格遵守

## 【必须执行】关键步骤

### 1. 策略多样化（至少4种）
- [ ] 符号回归 (Symbolic Regression)
- [ ] 遗传编程 (GP)
- [ ] 强化学习 (RL)
- [ ] LLM启发式生成

### 2. 技能感知反思（不可跳过）
- [ ] 成功因子 → 发现 / 优化
- [ ] 失败因子 → 技能缺陷 / 执行失误
- [ ] 每条轨迹最多3个反思
- [ ] 达到10个反思后整合修订

### 3. 独立审计门控（不可跳过）
- [ ] **数据泄露检测**：确认因子无未来函数
- [ ] **过拟合检测**：样本外 IC 必须达标
- [ ] **静默绕过检测**：确认技能被实际调用
- [ ] **部署失败检测**：实盘模拟无异常
- [ ] **功能完整性**：检查清单 100% 通过

## 【常见失误】执行失误警示

### ❌ 失误1：策略单一化
**后果**：所有因子陷入相同局部最优，缺乏多样性
**修正**：确保至少4种不同策略并行探索

### ❌ 失误2：跳过技能感知反思
**后果**：无法区分技能缺陷与执行失误，错误修改有效内容
**修正**：严格执行四种反思类型分类

### ❌ 失误3：跳过审计门控
**后果**：部署后才发现过拟合/未来函数
**修正**：严格执行五检查清单

### ❌ 失误4：混淆技能缺陷与执行失误
**后果**：错误修改有效技能内容 / 忽略真实技能问题
**修正**：仔细分析失败根因

### ❌ 失误5：静默绕过未检测
**后果**：技能存在但从未被使用
**修正**：审计时确认技能被实际调用

### ❌ 失误6：污染控制失效
**后果**：测试信息泄露到训练过程
**修正**：严格执行训练/测试分割

### ❌ 失误7：仅依赖样本内绩效
**后果**：选择过拟合因子，样本外表现差
**修正**：严格执行样本外验证

## 【强调标记】关键约束

> ⚠️ **警告**：以下约束必须严格遵守，否则可能导致严重后果

1. **禁止**策略单一化（少于4种策略）
2. **禁止**跳过技能感知反思阶段
3. **禁止**跳过独立审计阶段
4. **禁止**在训练期使用测试期数据
5. **禁止**生成包含未来函数的因子
6. **禁止**混淆主体修订与附录更新
7. **禁止**仅依赖样本内绩效选择因子
