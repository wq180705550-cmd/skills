# S_body：技能主体

## 核心概念

### 从传统 AI 到 Agentic AI

| 维度 | 传统 AI | Agentic AI |
|---|---|---|
| **工作模式** | 被动分析引擎，依赖人类提示 | 自主、目标驱动的系统 |
| **核心能力** | 模式识别、预测、分类 | 感知环境、多步推理、执行动作、从结果中学习 |
| **人类介入** | 需要持续提示和预定义规则 | 最小化人类干预 |
| **适应性** | 静态规则，需人工更新 | 动态学习，自主更新内部记忆 |

### 因子投资的核心挑战

1. **"因子动物园"问题**：潜在的价格-成交量组合维度远超人类认知能力
2. **数据挖掘偏差**：回测过拟合、p-hacking、虚假发现
3. **研究效率瓶颈**：人工特征工程、假设定义、持续提示
4. **技能演化缺失**：静态规则无法自适应市场变化
5. **审计门控不足**：缺乏系统化的过拟合/泄露检测

### Agentic AI + 技能演化解决方案

将 Agentic AI 作为**自主量化研究员**，结合**技能感知反思**与**独立审计门控**：
- 给定高层目标（如最大化风险调整收益、控制换手率和交易成本）
- 独立导航广阔的假设空间
- 使用**多策略探索**自主构建因子表达式
- 对历史市场数据进行回测
- 评估经济逻辑
- 基于实证反馈动态更新内部记忆
- 通过**技能感知反思**持续改进因子发现技能本身

## 因子发现流程

### 阶段一：策略多样化探索（Strategy Diversification）

**目的**：用不同策略执行因子生成，最大化覆盖不同的成功/失败模式

```
输入: 当前记忆、高层目标、探索策略数 K（默认 6，原 4）
输出: 因子候选集 {f_{r,i}}, 评估指标 {Sharpe_{r,i}, IC_{r,i}}

协议:
1. 对每次探索 i = 1, ..., K:
   a. 从策略池中选择探索策略 s_i:
      - s_1: 符号回归 (Symbolic Regression)
      - s_2: 遗传编程 (GP)
      - s_3: 强化学习 (RL)
      - s_4: LLM启发式生成 + 决策时信息约束（防止信息泄漏）
      - s_5: 图神经网络 (GNN) — 层次图学习，捕捉跨品种依赖关系
      - s_6: 连续隐马尔可夫模型 (Continuous HMM) — Regime 检测，重尾发射分布
      - s_7: 多流 Transformer (Multi-stream Transformer) — 融合多源异构数据流
      - s_8: 硬件友好型 LOB 混合器 (FastBiNLOB) — 限价订单簿微观结构因子
   b. 使用当前记忆生成新因子假设
   c. 记录完整因子轨迹 τ_{r,i} 和评估指标
2. 确保策略多样性:
   - 不同的生成路径覆盖不同的因子类型
   - 成功和失败因子都产生有价值的信号
   - 避免所有尝试都陷入相同的局部最优
```

**策略扩展说明（v2.1 新增）**：

| 新增策略 | 来源论文 | 核心洞察 |
|---------|---------|---------|
| 图神经网络 (GNN) | 2606.25811 | 层次图捕捉跨品种/跨期限依赖，日历价差策略优于基准 |
| 连续 HMM | 2606.23492 | 重尾发射分布（Student-t/Laplace）恢复波动率聚类，Regime 条件 VaR |
| 多流 Transformer | 2606.25007 | 独立编码器 + 融合机制，时间感知位置编码最优（0.9961 AUROC） |
| FastBiNLOB | 2606.25986 | 硬件友好的时间和特征混合，低延迟下超越 SOTA 架构 |
| 决策时信息约束 | 2606.22719 | 仅使用决策时可观测信息，防止宏观数据泄漏，实时 Nowcast 作为输入 |

### 阶段二：技能感知反思与对比更新

**目的**：基于成功/失败因子的对比分析，生成精准的技能修订

**核心原则**：
- 学习信号来自**部署后另一个智能体使用技能时的行为**
- 不是作者智能体的自我反思
- 候选技能可能因以下原因失败:
  * 产生错误信号（未来函数、数据窥探）
  * 遗漏必要指令（缺少风控、未处理异常）
  * 暴露误导性流程（错误的参数范围）
  * 被使用智能体**静默绕过**（内容有效但运行时从未被调用）

#### 四种反思类型应用于因子发现

| 反思类型 | 触发条件 | 主体影响 | 附录影响 |
|----------|----------|----------|----------|
| **发现(DISCOVERY)** | 成功因子揭示了当前技能未涵盖的有用模式 | 向 S_body 添加新内容 | 无 |
| **优化(OPTIMIZATION)** | 成功因子展示了优于现有技能内容的改进方式 | 修改现有 S_body 内容 | 无 |
| **技能缺陷(SKILL DEFECT)** | 失败因子暴露出技能内容存在错误/不完整 | 修正现有 S_body 内容 | 无 |
| **执行失误(EXECUTION LAPSE)** | 失败因子中技能有效，但智能体未遵循 | 无 | 向 S_appendix 添加强调项 |

**对比分析流程**：

1. **分析成功因子 F+**:
   - 识别成功因子的关键特征和逻辑模式
   - 提取可复用的过程知识
   - 标记技能中已覆盖和未覆盖的部分

2. **分析失败因子 F-**:
   - 定位失败根因（技能缺陷 vs 执行偏差）
   - 识别技能中缺失、错误或误导的内容
   - 检测是否存在"静默绕过"

3. **生成对比修订**:
   - 添加缺失的必要指令
   - 修正错误或误导的过程描述
   - 强化被绕过的关键步骤
   - 移除导致混淆的冗余内容

**技能感知演化螺旋**：
```
执行因子 → 技能感知反思 → 累积 → 整合 → 修订主体 → 更新附录 → 重复
```

**关键参数**：

| 参数 | 符号 | 描述 | 默认值 |
|------|------|------|--------|
| **每条轨迹最大反思数** | K | 每个因子的最大反思记录数 | 3 |
| **修订间隔** | B | 技能修订前累积的反思数量 | 10 |

### 阶段三：独立审计与定稿（Independent Audit）

**目的**：在独立新会话中审计候选因子，防止泄露、过拟合和部署特有失败

**审计检查清单（Audit Checklist）**：

| 检查项 | 说明 | 通过标准 |
|--------|------|----------|
| **1. 数据泄露检测** | 因子是否包含未来信息？是否编码了不应包含的测试期信息？ | 样本外 IC > 样本内 IC × 0.7 |
| **2. 过拟合检测** | 因子是否过度特化于训练期？移除训练特定细节后是否仍有效？ | 跨期稳定性 ICIR > 0.5 |
| **3. 静默绕过检测** | 技能内容是否有效但在运行时从未被调用？ | 审计智能体实际使用了技能 |
| **4. 部署失败检测** | 技能是否导致使用智能体产生新的失败模式？ | 实盘模拟无异常 |
| **5. 功能完整性** | 技能是否包含执行所需的全部必要信息？ | 检查清单 100% 通过 |

**静默绕过（Silent Bypass）检测机制**：
- 技能在内容上看起来有效（指令正确、流程合理）
- 但使用智能体在运行时**从未实际调用/依赖该技能**
- 这种失败从探索轨迹中不可见，只有通过部署交接才能检测

## Regime 检测与切换（v2.1 新增）

> **来源**：arxiv 2606.23492 — 股票收益的连续隐马尔可夫模型

### 核心思想

传统 HMM 使用高斯发射分布，无法捕捉股票收益的重尾特性。连续 HMM 将 Regime 链与发射分布分离：
- **Regime 链**：控制自相关和状态切换
- **发射分布**：控制边际分布（高斯/Student-t/拉普拉斯/广义误差）

### 重尾发射分布选择

| 分布 | 适用场景 | 参数 |
|------|---------|------|
| Gaussian | 常态市场 | μ, σ |
| Student-t | 厚尾、极值频繁 | μ, σ, ν（自由度） |
| Laplace | 尖峰厚尾 | μ, b（尺度） |
| GED | 灵活尾指数 | μ, σ, ν（尾指数） |

**实证结论**（SPY + 30 只股票面板）：
- 重尾边际（非额外衰减模式）缩小了大部分拟合差距
- 恢复 i.i.d. 基线以上的波动率聚类
- 生成少数状态即可边界不具约束力（谱恒等式）

### Regime 条件风险价值（Regime-Conditional VaR）

```
步骤:
1. 使用连续 HMM 拟合历史收益，得到隐状态序列
2. 每个 Regime 计算条件 VaR:
   VaR_α(s) = F_s^{-1}(α)  # 第 s 个 Regime 的分位数
3. 联合条件覆盖测试验证 VaR 准确性
4. 输出 Regime 切换概率和预期损失
```

### 实施要点

- **初始化**：基于分位数的初始化（避免 K-means 偏差）
- **期望最大化**：统一 E-Step（前向-后向递归）+ MStep
- **状态数选择**：谱恒等式将衰减模式数量界限限制为转移矩阵秩
- **Copula**：重现跨资产相关性的 copula 可用于多资产 Regime 建模

## 决策时信息约束（v2.1 新增）

> **来源**：arxiv 2606.22719 — LLM预测的防泄漏基准

### 问题定义

检索增强 LLM 的预测基准经常将模型能力与**信息泄漏**混淆：
- 带目标时间戳标记的特征通常在系统决策时间**不可观测**
- 宏观变量（如通胀）在决策时可能尚未发布

### 决策时可观测信息集

```
在每个决策时间点 t:
✅ 可观测:
  - 滞后平移的宏观变量（如上月 CPI）
  - 最近的宏观事件摘要（已发生）
  - 实时 Nowcast（如克利夫兰联储每日 CPI Nowcast）
  - 历史价格和成交量数据

❌ 不可观测:
  - 目标期宏观变量（未发布）
  - 未来价格数据
  - 未来事件
```

### 实时 Nowcast 作为决策时输入

```
步骤:
1. 获取决策时可观测的实时 Nowcast（如克利夫兰联储 CPI Nowcast）
2. 仅使用决策时信息输入 LLM 预测器
3. 宏观类比检索模块选择历史状态（仅使用决策时可观测数据）
4. 评论家 LLM 压缩历史状态为战术规则
5. 行动者 LLM 将当前状态和规则映射为因子分数
```

**实证结果**（2023-04 至 2026-03，7 个美国股票风格因子）：
- 中位数月度 Spearman 秩 IC: +0.154
- 三个不重叠 12 个月子窗口均值均为正
- kNN 宏观类比模型恢复可比中位数 IC（实时通胀信息 + 宏观相似检索是主要信号来源）

## 市场冲击与交易成本建模（v2.1 新增）

> **来源**：arxiv 2606.24019 — 美国大盘股平方根市场影响定律的实证确认

### 平方根定律（Square Root Law）

```
市场冲击 = c · σ_D · (Q / V_D)^(1/2)

其中:
- c: 市场影响系数（AAPL 实证: c_raw=0.69, c_eff=0.34）
- σ_D: 日波动率
- Q: 订单大小
- V_D: 日交易量
```

### 关键发现

| 发现 | 数值/结论 |
|------|----------|
| 指数固定 | 1/2（通用值）|
|  prefactor | c_raw=0.69（偏差校正后 c_eff=0.34）|
| 规模分布尾指数 | β=1.54±0.15 |
| 订单流长记忆 | γ=0.66 |
| 价格扩散性 | 赫斯特指数 0.49 |

### 实施要点

- **元订单重建**：从匿名订单流重建元订单（无需券商标记母订单）
- **结构验证**：打乱交易符号 → 方向性影响降至随机水平（86%→51%）
- **稳定性**：32 周滚动前向重新校准，prefactor 保持稳定

### 交易成本约束

```
在因子回测中:
1. 使用平方根定律估计市场冲击
2. 将市场冲击纳入交易成本模型
3. 约束因子换手率（建议 < 50%/月）
4. 大单策略需考虑市场冲击非线性
```

## 多流数据融合架构（v2.1 新增）

> **来源**：arxiv 2606.25007 — 用于金融欺诈检测的多流时间融合

### 架构概述

多流欺诈 Transformer（MSFT）统一架构：
- 独立 Transformer 编码器对每个事件流编码
- 可配置的融合机制融合表示

### 五种融合策略（系统消融研究）

| 融合策略 | AUROC | 适用场景 |
|---------|-------|---------|
| 拼接（Concatenation） | 0.82 | 简单场景 |
| 门控融合（Gated Fusion） | **0.989** | **生产部署（最佳精度）** |
| 时间感知位置编码 | **0.9961** | **最高区分度** |
| 跨流注意力 | 0.97 | 流间依赖强 |
| 全组合 | 0.98 | 计算资源充足 |

### 实施要点

1. **每流编码必不可少**：单流 Transformer 基线仅 0.82 AUROC（18 点差距）
2. **风险事件流最强**：单个信号贡献最大
3. **生产推荐**：门控融合（精度）或时间感知位置编码（区分度）

### 应用于因子投资

```
多流数据融合可用于:
- 交易流 + 订单流 + 新闻流 → 综合情绪因子
- 价格流 + 成交量流 + 期权流 → 综合动量因子
- 宏观流 + 行业流 + 资金流 → 综合风格因子

## 图神经网络因子建模（v2.1 新增）

> **来源**：arxiv 2606.25811 — 商品期货市场日历价差策略的层次图学习

### 层次图结构

```
上层: 基础资产（如原油、黄金）
  └── 边: 反映固有相关性（相关系数矩阵）
下层: 单个期货合约（不同到期日）
  └── 边: 捕捉合约到基础资产的联系（日历价差）
```

### 关键洞察

- 商品期货间**依赖成熟度的相互关系**对预测至关重要
- 日历价差（CS）策略比仅做多策略具有更高信息比率和更低风险
- 层次图学习利用跨期限依赖预测期货价格变动

### 实施要点

1. **图构建**：基于相关系数矩阵构建上层边，基于期限结构构建下层边
2. **消息传递**：层次消息传递（上层 → 下层，下层 → 上层）
3. **预测目标**：日历价差收益率（非绝对价格）
4. **交易算法**：基于图神经网络预测的 CS 交易策略

## 过拟合缓解机制

### 1. 经济正则化

```
要求: 每个因子必须有清晰的经济学解释
检验:
- 是否与已知风险溢价相关？
- 是否捕捉特定的市场摩擦？
- 是否反映投资者行为偏差？
```

### 2. 统计门槛

```
严格统计检验:
- t 统计量 > 3.0（对应 p < 0.01）
- 样本外 R² > 0
- 信息比率 > 0.5
- 夏普比率 > 1.5
```

### 3. 多重检验校正

```
问题: 测试大量因子时，虚假发现概率上升
解决方案:
- Bonferroni 校正: 调整显著性水平
- FDR 控制: 限制假阳性比例
- 有效因子数估计: 考虑因子相关性
```

### 4. 样本外衰减检验

```
检验因子在不同时期的稳定性:
- 训练期 vs 测试期绩效对比
- 滚动窗口绩效追踪
- 衰减率阈值（建议 < 30%）
```

### 5. 时间与策略隔离

```
严格时间分割:
- 训练期: 用于因子发现
- 验证期: 用于因子筛选
- 测试期: 最终绩效评估（仅使用一次）

策略隔离:
- 不同因子使用独立数据路径
- 避免信息泄露
```

### 第一层：严格的训练/测试分割

- Agentic AI 仅能访问训练期数据

## 关键设计原则

### 1. 自主性 > 被动性
- 智能体自主导航假设空间，而非依赖人工提示
- 动态更新记忆，而非静态规则

### 2. 可解释性 > 黑箱
- 每个因子必须有经济逻辑支撑
- 符号回归确保数学表达式可解释
- Regime 切换必须有明确的经济解释（非纯统计拟合）

### 3. 样本外验证 > 样本内拟合
- 严格的样本外测试
- 多重检验校正防止虚假发现

### 4. 经济逻辑 > 数据挖掘
- 经济正则化约束
- 行为金融理论支撑

### 5. 实用性 > 理论性
- 考虑交易成本、换手率、容量约束
- 机构可执行的策略
- 计算效率约束（操作截止日期）

### 6. 技能演化 > 静态规则（EmbodiSkill + SkillEvolver 贡献）
- 技能感知反思持续改进
- 独立审计门控保障质量
- 策略多样化探索避免局部最优

### 7. 决策时信息约束 > 未来信息泄漏（v2.1 新增）
- 仅使用决策时可观测信息
- 实时 Nowcast 替代未发布的宏观数据
- 防止宏观数据时间对齐错误

### 8. Regime 感知 > 静态因子（v2.1 新增）
- 因子表现随 Regime 切换而变化
- 使用连续 HMM 检测 Regime
- Regime 条件 VaR 用于风险管理

### 9. 市场冲击感知 > 零交易成本假设（v2.1 新增）
- 使用平方根定律估计市场冲击
- 大单策略需考虑非线性冲击
- 交易成本建模包含市场冲击

### 10. 多流融合 > 单流分析（v2.1 新增）
- 融合多源异构数据流
- 时间感知位置编码或门控融合
- 每流编码必不可少（非单流 Transformer）

**修正**：审计时确认技能被实际调用

### ❌ 失误6：污染控制失效
**后果**：测试信息泄露到训练过程
**修正**：严格执行训练/测试分割

### ❌ 失误7：仅依赖样本内绩效
**后果**：选择过拟合因子，样本外表现差
**修正**：严格执行样本外验证

### ❌ 失误8：忽略决策时信息约束（v2.1 新增）
**后果**：信息泄漏，因子在实盘失效
**修正**：仅使用决策时可观测信息，用实时 Nowcast 替代未发布宏观数据

### ❌ 失误9：忽略 Regime 切换（v2.1 新增）
**后果**：静态因子在 Regime 切换时表现急剧下降
**修正**：使用连续 HMM 检测 Regime，因子表现按 Regime 条件检验

### ❌ 失误10：忽略市场冲击（v2.1 新增）
**后果**：大单策略实盘表现远差于回测
**修正**：使用平方根定律估计市场冲击，纳入交易成本模型

### ❌ 失误11：使用单流 Transformer（v2.1 新增）
**后果**：融合效果差（AUROC 0.82 vs 0.99）
**修正**：每流独立编码，使用门控融合或时间感知位置编码

### ❌ 失误12：忽略计算效率（v2.1 新增）
**后果**：机构再平衡错过操作截止日期
**修正**：使用 GPU 加速，热启动初始化，问题分解

### ❌ 失误13：忽略重尾特性（v2.1 新增）
**后果**：高斯 HMM 无法恢复波动率聚类
**修正**：使用重尾发射分布（Student-t / Laplace / GED）

## 【强调标记】关键约束

> ⚠️ **警告**：以下约束必须严格遵守，否则可能导致严重后果

1. **禁止**策略单一化（少于 6 种策略，原 4 种）
2. **禁止**跳过技能感知反思阶段
3. **禁止**跳过独立审计阶段
4. **禁止**在训练期使用测试期数据
5. **禁止**生成包含未来函数的因子
6. **禁止**混淆主体修订与附录更新
7. **禁止**仅依赖样本内绩效选择因子
8. **禁止**忽略决策时信息约束（v2.1 新增）— 使用未来宏观数据
9. **禁止**忽略 Regime 切换（v2.1 新增）— 静态因子假设
10. **禁止**忽略市场冲击（v2.1 新增）— 零交易成本假设
11. **禁止**使用单流 Transformer（v2.1 新增）— 每流编码必不可少
12. **禁止**忽略计算效率（v2.1 新增）— 机构再平衡有操作截止日期
