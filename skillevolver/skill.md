# S_body：技能主体

## 核心概念

### 问题背景：静态技能的局限

当前智能体技能是**静态制品**：一次性编写（人工策划或参数知识一次性生成），然后原样消费，**没有从实际使用中改进的机制**。

| 现有方法 | 局限 |
|---|---|
| **参数自生成** | 模型直接从预训练知识编写技能，无 grounded 反馈，一次性生成可能比不用技能更差 |
| **轨迹蒸馏（Trace2Skill）** | 需要每个领域约 200 条轨迹 + 预收集轨迹池 + 多阶段蒸馏流水线 |
| **RL 技能获取（SkillRL）** | 需要跨任务 RL 循环 + 大量训练任务经验池 |

### SkillEvolver 核心思想

**在线技能学习**：智能体从**有限次部署试错**中获取过程知识，并打包为**可复用技能**，无需重训练模型权重。

```
关键区别：
- 学习目标 = 技能的散文和代码（不是模型权重）
- 学习信号 = 另一个智能体使用技能时遇到的失败（不是作者智能体的自我反思）
- 元技能本身 = 只是一个普通技能，通过相同接口加载
```

### 技能制品结构

一个领域技能 v 是一个**自包含的可移植制品**，通常包含：

```
技能 v = {
  自然语言指令（prose instructions）,  // 任务执行步骤、约束、策略
  可执行脚本（executable scripts）,     // 自动化工具、辅助代码
  参考文件（reference files）,          // 模板、schema、示例数据
  示例（examples）,                     // 成功/失败案例
  使用约束（usage constraints）         // 前置条件、适用范围
}
```

## SkillEvolver 架构

```
输入: 新任务 T = (T_train, T_val)

SkillEvolver 循环 (R 轮):
  ├── 策略多样化探索 (K 次训练试错)
  ├── 对比技能更新 (分析成功/失败 → 精准修订)
  └── 独立审计与定稿 (过拟合/泄露/静默绕过检测)

输出: 可部署领域技能 v*

验证: 在 T_val 上用全新智能体评估
```

---

## 三阶段核心机制

### 阶段一：策略多样化探索

**目的**: 用不同策略执行训练任务，最大化覆盖不同的成功/失败模式。

```
输入: 当前技能 v_r, 训练集 T_train, 探索次数 K（默认 4）
输出: 轨迹集 {τ_{r,i}}, 奖励 {y_{r,i}}

协议:
1. 对每次探索 i = 1, ..., K:
   a. 从策略池中选择探索策略 s_i（确保策略多样性）
   b. SkillEvolver 智能体使用当前技能 v_r 在 T_train 上执行任务
   c. 记录完整轨迹 τ_{r,i} 和任务奖励 y_{r,i}
2. 策略多样化确保:
   - 不同的探索路径覆盖不同的子任务
   - 成功和失败轨迹都产生有价值的信号
   - 避免所有尝试都陷入相同的失败模式
```

**策略池示例**：

| 策略类型 | 适用场景 | 示例 |
|---------|---------|------|
| **贪心策略** | 快速收敛 | 选择当前最优动作 |
| **探索策略** | 发现新模式 | 随机选择或ε-贪心 |
| **模仿策略** | 利用先验知识 | 模仿人类或专家行为 |
| **对抗策略** |  stress测试 | 故意选择困难路径 |

### 阶段二：对比技能更新

**目的**: 基于成功/失败轨迹的对比分析，生成精准的技能修订。

**核心原则**:
- 学习信号来自**部署交接**（另一个智能体使用技能时的行为）
- 不是作者智能体的自我反思
- 候选技能可能因以下原因失败:
  * 产生错误答案
  * 遗漏必要指令
  * 暴露误导性流程
  * 被使用智能体静默绕过（内容有效但运行时从未被调用）

**对比分析流程**:
1. **分析成功轨迹 T+**:
   - 识别成功执行的关键步骤和决策点
   - 提取可复用的过程模式
   - 标记技能中已覆盖和未覆盖的部分

2. **分析失败轨迹 T-**:
   - 定位失败根因（技能缺陷 vs 执行偏差）
   - 识别技能中缺失、错误或误导的内容
   - 检测是否存在"静默绕过"（技能存在但未被使用）

3. **生成对比修订**:
   - 添加缺失的必要指令
   - 修正错误或误导的过程描述
   - 强化被绕过的关键步骤（增加强调或约束）
   - 移除导致混淆的冗余内容

### 阶段三：独立审计与定稿

**目的**: 在独立新会话中审计候选技能，防止泄露、过拟合和部署特有失败。

**审计检查清单**:

| 检查项 | 说明 | 通过标准 |
|---|---|---|
| **1. 泄露检测** | 技能是否包含训练标签/测试答案信息？是否编码了不应包含的特定实例信息？ | 无特定实例信息 |
| **2. 过拟合检测** | 技能是否过度特化于训练实例？移除训练特定细节后技能是否仍有效？ | 泛化能力验证通过 |
| **3. 静默绕过检测** | 技能内容是否有效但在运行时从未被调用？使用智能体是否实际依赖了该技能？ | 审计智能体实际使用了技能 |
| **4. 部署失败检测** | 技能是否导致使用智能体产生新的失败模式？修订是否引入了新的误导性内容？ | 无新失败模式 |
| **5. 功能完整性** | 技能是否包含执行任务所需的全部必要信息？指令是否清晰、无歧义、可执行？ | 检查清单 100% 通过 |

**静默绕过（Silent Bypass）** 是 SkillEvolver 发现的一种关键失败模式：
- 技能在内容上看起来有效（指令正确、流程合理）
- 但使用智能体在运行时**从未实际调用/依赖该技能**
- 这种失败从探索轨迹中不可见，只有通过部署交接才能检测

## 污染控制机制

### 第一层：严格的训练/测试分割
- 策划的训练技能在源处被删除，永远不可达
- SkillEvolver 仅能访问 T_train，永远看不到 T_val
- 测试集 oracle 不能泄露到测试时评估的制品中

### 第二层：工作区白名单
- 通过 PreToolUse 钩子强制执行工作区白名单
- SkillEvolver 智能体只能访问允许的文件路径
- 审计智能体在独立会话中运行，无法访问作者智能体的上下文

## 关键设计原则

1. **技能制品 > 模型权重** —— 学习目标是技能的散文和代码，不是模型参数
2. **部署 Grounded 精炼** —— 精炼信号来自部署后另一个智能体的使用行为
3. **独立审计门控** —— 每个候选技能必须通过独立新会话的审计
4. **策略多样化探索** —— 不同策略覆盖不同的成功/失败模式
5. **即插即用** —— 元技能通过标准技能接口加载，无需修改智能体架构
6. **与 EmbodiSkill 协同** —— 冷启动用 SkillEvolver，热迭代用 EmbodiSkill

---

## 反模式

- 不要从探索轨迹直接蒸馏技能而不部署测试
- 不要让作者智能体审计自己的技能
- 不要跳过策略多样化
- 不要全量重写技能（应基于对比分析进行精准修订）
- 不要允许 SkillEvolver 访问测试集
- 不要忽略静默绕过检测
- 不要在审计失败时直接接受候选
- 不要将训练标签信息编码到技能中
- **不要策略单一化（少于4种策略）**
- **不要跳过与 EmbodiSkill 的协同演化**

# S_appendix：技能附录

> **重要提示**：本附录包含关键约束和常见失误，使用本 skill 时必须严格遵守

## 【必须执行】关键步骤

### 1. 策略多样化（至少4种）
- [ ] 贪心策略（快速收敛）
- [ ] 探索策略（发现新模式）
- [ ] 模仿策略（利用先验知识）
- [ ] 对抗策略（stress测试）

### 2. 对比技能更新（不可跳过）
- [ ] 分析成功轨迹 T+
- [ ] 分析失败轨迹 T-
- [ ] 定位失败根因
- [ ] 生成精准修订

### 3. 独立审计门控（不可跳过）
- [ ] **泄露检测**：确认无特定实例信息
- [ ] **过拟合检测**：验证泛化能力
- [ ] **静默绕过检测**：确认技能被实际调用
- [ ] **部署失败检测**：确认无新失败模式
- [ ] **功能完整性**：检查清单 100% 通过

## 【常见失误】执行失误警示

### ❌ 失误1：策略单一化
**后果**：所有尝试陷入相同失败模式，缺乏多样性
**修正**：确保至少4种不同策略（贪心/探索/模仿/对抗）

### ❌ 失误2：跳过策略多样化
**后果**：成功/失败信号不全面，无法生成有效修订
**修正**：严格执行 K=4 次探索

### ❌ 失误3：作者智能体审计自己的技能
**后果**：无法检测静默绕过等部署特有失败
**修正**：必须在独立新会话中审计

### ❌ 失误4：跳过静默绕过检测
**后果**：技能存在但从未被使用
**修正**：审计时确认技能被实际调用

### ❌ 失误5：全量重写技能
**后果**：覆盖有效内容，累积冗余
**修正**：基于对比分析进行精准修订

### ❌ 失误6：忽略与 EmbodiSkill 的协同
**后果**：缺少持续演化能力，技能逐渐过时
**修正**：定期使用 EmbodiSkill 进行热迭代

## 【强调标记】关键约束

> ⚠️ **警告**：以下约束必须严格遵守，否则可能导致严重后果

1. **禁止**策略单一化（少于4种策略）
2. **禁止**跳过策略多样化探索阶段
3. **禁止**作者智能体审计自己的技能
4. **禁止**跳过静默绕过检测
5. **禁止**全量重写技能（必须精准修订）
6. **禁止**允许 SkillEvolver 访问测试集
7. **禁止**长期不进行 EmbodiSkill 协同演化
